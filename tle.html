Lesson 1: Cascading Style Sheet (CSS)

What is CSS?

Cascading Style Sheet

Describes how HTML elements are to be displayed on media

Saves a lot of work

External stylesheets are stored in CSS files

CSS is the language we use to style a web page

Used to define styles for your web pages

You should have a basic understanding of HTML or XHTML before learning CSS

Layers of the Webpage

Content (HTML)

Presentation (CSS)

Behavior (JavaScript)

Content Layer

Defines and lays out the text, images, animation, sound, and video on a web page using tags.

  • Text must be easy to understand and make sense
  • Include alternative text
  • Ensure content fits for the web
  • Explain acronyms and abbreviations
  • Make images unambiguous for colorblind users
  • Provide users with necessary environmental changes

Presentation Layer

Defines how people see your web page in a browser.

  • Ensure text can be zoomed
  • Make interactive elements easy to find
  • Ensure contrast and clarity for colorblind users
  • Give consistent navigation
  • Aid users through business processes
  • Separate content into understandable units

Behavior Layer

Where real-time user interaction happens.

  • Ensure all functionality is available regardless of input device
  • Make the user experience as easy as possible

HTML vs CSS

HTML: Creates the actual content or foundation of the page

CSS: Responsible for the design or style of the website, including layout, effects, and colors

Benefits of Using CSS

  • Saves a lot of work
  • Easy maintenance
  • Makes web pages more visually appealing

Capabilities of CSS

  • Makes pages easily updatable
  • Positions objects on the page
  • Layers objects on the page
  • Creates custom tags

Advantages of Using CSS

  • Saves typing and development time
  • Downloads faster since the browser only loads one file once
  • Allows multiple linked CSS files in one document

Lesson 2: Style Sheet and Its Parts

What is a Style Sheet?

A set of instructions to a Web browser on how to display various elements on a Web page is known as a style sheet. Every CSS is a series of instructions called statements that:

  • Identify the elements in an HTML document that it affects
  • Tell the browser how to draw these elements
  • Apply to elements such as paragraphs, links, and list items
  • Are just text files or text embedded in the head of an HTML document that separate content from appearance
  • Suggest to the browser how the page should be displayed

How CSS is Applied

Linked method: CSS is stored in a separate file, instead of directly in the HTML page.

Embedded method: CSS is stored as part of the HTML page, in the header section.

Inline method: CSS is stored directly in the style attributes of HTML tags.

Inline Style

You can define the style for a single element using the style attribute. Various HTML tags can have the style attribute, and its value defines how that line will appear.

Embedded Style

Defined within the <style> tags inside the head part of an HTML file. It provides style for the entire HTML document.

Linked Style

Linked CSS is made outside the HTML file using the <link> element. It establishes a relationship between the HTML document and the stylesheet file.

Used only within the <head></head> section

The stylesheet file contains only style rules, no HTML code

Creating an External CSS

  1. Open a blank document in Notepad.
  2. Type the styles you want to define.
  3. On the Menu Bar, click File → Save.
  4. Type your filename with a .css extension and select All Files as the file type.
  5. Linking Your CSS to an HTML File: After creating your CSS file, open your HTML file and type this inside the head section:

<link rel="stylesheet" href="linked.css" type="text/css">

Then save your HTML file.

Lesson 3: Display Property

Definition

The display property is one of the most important CSS properties for controlling layout. It determines how an element is shown on a web page.

Main Display Values

  • block — starts on a new line and takes up the full width available (e.g. <div>, <p>, <h1>–<h6>)
  • inline — does not start on a new line; only takes up as much width as necessary (e.g. <span>, <a>)
  • inline-block — behaves like inline but allows setting width and height
  • none — hides the element completely; it will not occupy any space

Examples

display: block; — Example: <div>, <header>

display: inline; — Example: <span>, <a>

display: inline-block; — Example: buttons or icons in one row

display: none; — used to hide an element dynamically

Additional Display Values

  • flex — enables flexible layouts with items aligned in rows or columns
  • grid — allows arranging items into rows and columns simultaneously
  • table — behaves like an HTML table element
  • list-item — behaves like a list element (<li>)

Example CSS


div {
  display: flex;
  justify-content: center;
  align-items: center;
}

Summary

By changing an element’s display property, you can control how elements appear, stack, or flow in a layout — essential for responsive design.

Lesson 4: CSS Rule

Definition

A CSS rule defines how to style an HTML element. Each rule consists of a selector and a declaration block.

Structure of a CSS Rule


selector {
  property: value;
}

Example:


p {
  color: blue;
  font-size: 16px;
}

Explanation

  • Selector — indicates which HTML element is affected (e.g. p for paragraphs)
  • Property — what aspect of the element you want to change (e.g. color, font-size)
  • Value — defines how that property should look (e.g. blue, 16px)

Types of CSS Rules

  • Element Selector — targets HTML tags directly (e.g. h1 {color:red;})
  • Class Selector — targets elements with a specific class (e.g. .main {text-align:center;})
  • ID Selector — targets a unique element (e.g. #footer {background:black;})
  • Group Selector — combines multiple selectors (e.g. h1, h2, h3 {font-weight:bold;})
  • Universal Selector — applies to all elements (e.g. * {margin:0;})

How a Browser Reads CSS Rules

  1. The browser finds the selector and matches it with HTML elements.
  2. It then applies each property–value pair within the declaration block.
  3. If there are conflicting styles, the browser uses the rule of specificity and cascading.

Lesson 5: CSS Selectors

Definition

Selectors in CSS are patterns used to select the element(s) you want to style.

Basic Selectors

  • Type Selector — selects all elements of a given type (e.g. p)
  • Class Selector — selects all elements with a given class (e.g. .menu)
  • ID Selector — selects a unique element with a given id (e.g. #header)
  • Universal Selector — selects all elements (e.g. *)

Grouping Selectors

Multiple selectors can share the same style rules using commas:


h1, h2, h3 {
  font-family: Arial, sans-serif;
}

Combinator Selectors

  • Descendant Selector — div p selects all p inside a div
  • Child Selector — div > p selects all p that are direct children of div
  • Adjacent Sibling — h1 + p selects the p immediately after h1
  • General Sibling — h1 ~ p selects all p siblings after h1

Pseudo-classes

Select elements based on a state:

  • a:hover — when the link is hovered
  • li:first-child — first list item
  • input:focus — focused input

Lesson 6: CSS Dimensions

Introduction

CSS dimensions define the size of elements on a web page. You can set width, height, margin, padding, and border.

Width and Height

  • Width: the content width of an element
  • Height: the content height of an element

Example:


div {
  width: 300px;
  height: 150px;
}

Margin

Margin creates space outside an element.


p {
  margin: 10px 20px;
}

Padding

Padding creates space inside an element, between content and border.


div {
  padding: 15px;
}

Border

Adds a visible line around an element.


div {
  border: 2px solid black;
}

Box Model

The CSS Box Model describes how width and height are calculated including content, padding, border, and margin.

PropertyDescription
ContentThe actual content inside the element
PaddingSpace between content and border
BorderThe border surrounding the padding
MarginSpace outside the element

Units

Dimensions can be set using:

  • px — pixels
  • % — percentage of parent element
  • em/rem — relative to font size
  • vh/vw — relative to viewport height/width

Definition

Font Families (font-family)

Quarter 4 — Lesson 1

Introduction to JavaScript

JavaScript (JS) is the primary language used to make websites interactive. While HTML provides the structure and CSS provides the style, JS provides the behavior.

1. Key Concepts and Origin

  • The Founder: Created by Brendan Eich at Netscape in 1995. It was developed in only 10 days.
  • Original Names: First called Mocha, then LiveScript, before finally being marketed as JavaScript to capitalize on the popularity of the Java language.

Nature of JS:

  • Client-side: The script runs on the user's browser, reducing the load on the web server.
  • High-level: Uses syntax that is easy for humans to read and write.
  • Interpreted: Browsers read the code directly; it doesn't need to be "compiled" into a separate file first.
  • Object-Oriented: It treats data and functions as "objects" that can be manipulated.
  • DHTML: Combining JS with CSS and HTML creates Dynamic HTML, which allows page content to change without a full refresh.

2. Advantages & Disadvantages

Pros:

  • Lightweight and fast.
  • Works on all modern browsers (Chrome, Safari, Edge, Firefox).
  • License-free: You don't have to pay to use the language.

Cons:

  • Security: Malicious scripts can be hidden in pages.
  • Exposure: Since it is client-side, anyone can "View Source" and see your logic.
  • Browser Consistency: Some older mobile devices or browsers might interpret JS differently, leading to errors.

Quarter 4 — Lesson 2

Script Placement and Guidelines

To use JavaScript, you must tell the browser where the code starts and ends using the <script> tag.

1. Where to Place Your Code

  • <head> Section: Scripts here are pre-loaded. This is the best place for functions that need to be ready as soon as the user interacts with the page.
  • <body> Section: Usually placed right before the closing </body> tag. This ensures the HTML content loads first, so the user doesn't see a blank page while the script is loading.
  • External JS: You write code in a separate file (e.g., script.js) and link it using <script src="script.js"></script>. This is best for keeping your project organized.

2. Coding Guidelines

  • Case Sensitivity: JavaScript is very strict. myVariable and myvariable are treated as two different things.
  • White Space: Extra spaces or tabs between code are ignored by JS, but they are vital for humans to read the code easily.
  • Comments: Use // for single-line comments and /* ... */ for multi-line comments. These are ignored by the browser.
  • Syntax Tip: Always use the type attribute to be safe: <script type="text/JavaScript">.

Quarter 4 — Lesson 3

The Building Blocks (Objects and Variables)

JavaScript views everything on your screen as an Object.

1. Core Components

  • Objects: Items in the browser (e.g., the window, the document, a button).
  • Methods: These are actions. Think of them as verbs (e.g., document.write() tells the document to write text).
  • Properties: These are characteristics. Think of them as adjectives (e.g., document.bgColor describes the background color).
  • Parameters: Extra information passed into a method (e.g., in alert("Hello"), "Hello" is the parameter).

2. Variables and Data Types

Variables are containers for storing data. You declare them using the var keyword.

Rules:

  • Must start with a letter or underscore (never a number).
  • No spaces or punctuation.
  • Names are case-sensitive.

Data Types:

  • String: Text inside quotes ("Hello").
  • Number: Integers (10) or Decimals (10.5).
  • Boolean: Logical true or false.
  • Null/Undefined: Values for when a variable is empty or hasn't been set yet.

Quarter 4 — Lesson 4

Events, Popups, and Control Flow

1. Event Handlers

Events are triggers that make code run.

  • Mouse Events: onclick, ondblclick, onmouseover (hovering), onmouseout.
  • Keyboard Events: onkeydown, onkeyup.
  • Form/Page Events: onload (when page finishes loading), onsubmit (when a form is sent), onblur (when an input field loses focus).

2. Interactive Popup Boxes

  • Alert: alert("Warning!"); — Only has an OK button. Used for announcements.
  • Confirm: confirm("Are you sure?"); — Has OK and Cancel. Returns true or false.
  • Prompt: prompt("Enter name:", "Guest"); — Provides a text box for the user to type in.

3. Making Decisions (Conditionals)

Control flow allows your program to make choices based on data.

  • if statement: Runs code ONLY if the condition is true.
  • else if: Checks a second condition if the first one was false.
  • else: The final fallback; runs if none of the above conditions were met.
  • switch: Used when you have many different specific cases to check against a single variable.
